# 需求规格文档 (SRS)

## 项目概述

- **项目名称**: PanoDB - HarmonyOS PC 数据库管理工具
- **项目类型**: HarmonyOS PC 原生应用
- **开发平台**: OpenHarmony 6.0 (API Level 20)
- **文档版本**: v1.1
- **创建日期**: 2026-03-26

---

## 1. 引言

### 1.1 目的

本文档旨在详细定义 PanoDB 项目的功能需求、非功能需求及用户界面要求，为开发团队提供明确的开发依据，并为后续的测试和验收工作提供参考标准。

### 1.2 范围

本文档涵盖以下范围：
- PanoDB 应用程序的核心功能需求
- 数据库管理基本操作的需求定义
- 用户界面交互需求
- 性能、安全性等非功能需求
- 数据定义与接口需求

### 1.3 定义与缩略语

| 术语 | 定义 |
|------|------|
| PanoDB | 本项目的数据库管理工具名称 |
| RDB Store | OpenHarmony 的关系型数据库存储组件 (@ohos.data.rdb) |
| ArkTS | OpenHarmony 的应用开发语言 (TypeScript 超集) |
| ArkUI | OpenHarmony 的声明式 UI 框架 |
| SQLite | 轻量级嵌入式关系型数据库 |
| Schema | 数据库结构定义 (表、列、索引、视图等) |

## 2. 总体描述

### 2.1 产品背景

随着鸿蒙生态的快速发展，HarmonyOS PC 作为新兴的桌面操作系统，急需配套的数据库管理工具。目前市面上缺乏专为 HarmonyOS PC 设计的原生数据库管理应用，用户只能使用命令行工具或跨平台方案。本项目旨在填补这一生态空白，提供一款流畅、专业的原生数据库管理工具。

### 2.2 产品定位

**目标用户画像**:
- 移动应用开发者 (HarmonyOS 应用开发)
- 数据库初学者
- 需要轻量级数据库管理工具的用户
- 数据分析师 (处理本地 SQLite 数据)

**核心价值主张**:
- 原生体验: 专为 HarmonyOS PC 打造，充分利用系统特性
- 简单易用: 直观的界面设计，降低学习成本
- 轻量高效: 占用资源少，启动快速

### 2.3 用户操作流程

```
用户启动应用
    │
    ├── [新建数据库] → 输入名称 → 创建成功 → 进入主界面
    │
    └── [打开数据库] → 选择文件 → 加载中... → 进入主界面
                              │
                              └── 主界面功能 ──────────┐
                                    │                 │
                                    ├── 浏览 Schema ───┤
                                    ├── 执行 SQL   ────┤
                                    ├── 编辑数据   ────┤
                                    └── 导出结果   ────┘
```

### 2.4 约束条件

| 约束类型 | 描述 |
|----------|------|
| 平台约束 | 仅支持 OpenHarmony 6.0 (API Level 20) 及以上版本 |
| 设备约束 | 需支持 HarmonyOS PC 设备 |
| 存储约束 | 单个数据库文件大小受文件系统限制 |
| 网络约束 | 本版本不支持远程数据库连接 |

### 3.2 详细功能需求

#### 3.2.1 数据库管理模块 (F-DB-xxx)

##### F-DB-001: 新建数据库

| 属性 | 描述 |
|------|------|
| 功能编号 | F-DB-001 |
| 功能名称 | 新建数据库 |
| 功能描述 | 用户可以创建一个新的 SQLite 数据库文件 |
| 优先级 | P0 (核心功能) |
| 触发条件 | 用户点击"新建数据库"按钮 |
| 前置条件 | 无 |
| 基本流程 | 1. 用户点击"新建数据库"按钮<br>2. 系统弹出创建对话框<br>3. 用户输入数据库名称<br>4. 用户选择存储路径 (可选)<br>5. 用户点击"创建"按钮<br>6. 系统创建空数据库文件<br>7. 系统自动打开该数据库<br>8. 系统显示创建成功提示 |
| 异常处理 | - 如果名称为空，提示"请输入数据库名称"<br>- 如果名称包含非法字符，提示"名称不能包含特殊字符"<br>- 如果路径不可写，提示"无法在该位置创建文件" |
| 后置条件 | 新数据库被创建并打开 |

##### F-DB-002: 打开数据库 【已修订】

> **重要说明**：由于 HarmonyOS 应用沙箱隔离机制，应用无法直接访问外部文件。必须通过以下"选择→拷贝→操作→导出"流程。

| 属性 | 描述 |
|------|------|
| 功能编号 | F-DB-002 |
| 功能名称 | 打开数据库 |
| 功能描述 | 用户可以打开一个已存在的 SQLite 数据库文件（通过系统选择器选取后拷贝至应用沙箱） |
| 优先级 | P0 (核心功能) |
| 触发条件 | 用户点击"打开数据库"按钮 |
| 前置条件 | 目标 .db/.sqlite 文件存在 |
| 基本流程 | 1. 用户点击"打开数据库"按钮<br>2. 系统调用 DocumentViewPicker 打开系统文件选择器<br>3. 用户浏览并选择 .db/.sqlite 文件<br>4. 用户点击"确认"<br>5. 系统将文件**拷贝**到本应用 cacheDir 沙箱目录<br>6. 系统在沙箱副本上验证文件格式<br>7. 系统加载数据库 Schema 信息<br>8. 系统进入数据库主界面<br>9. [退出时提示] 是否导出到原始位置？ |
| 异常处理 | - 如果文件不存在，提示"文件不存在"<br>- 如果文件不是有效的 SQLite 数据库，提示"无效的数据库文件"<br>- 如果拷贝失败，提示"无法读取该文件"<br>- 如果应用存储空间不足，提示"存储空间不足" |
| 后置条件 | 数据库（沙箱副本）被加载，用户可以浏览和操作 |
| 文件同步 | **重要**：用户在应用内的所有修改仅作用于沙箱副本，需通过"导出"功能将修改写回原始位置 |

## 5. 数据定义

### 5.1 核心数据模型

#### 5.1.1 数据库信息

```typescript
interface DatabaseInfo {
  id: string;              // 唯一标识符
  name: string;           // 数据库名称
  path: string;           // 文件路径
  size: number;           // 文件大小 (字节)
  tableCount: number;     // 表数量
  createdTime: number;    // 创建时间 (时间戳)
  modifiedTime: number;   // 最后修改时间 (时间戳)
}
```

#### 5.1.2 表信息

```typescript
interface TableInfo {
  name: string;           // 表名
  type: 'table' | 'view'; // 类型
  rowCount: number;       // 行数
  columns: ColumnInfo[];  // 列信息
}
```

#### 5.1.3 列信息

```typescript
interface ColumnInfo {
  name: string;           // 列名
  dataType: string;       // 数据类型 (INTEGER, TEXT, REAL, BLOB)
  notNull: boolean;       // 是否非空
  primaryKey: boolean;    // 是否主键
  defaultValue?: string;  // 默认值
}
```

#### 5.1.4 查询结果

```typescript
interface QueryResult {
  success: boolean;      // 是否成功
  columns: string[];     // 列名列表
  rows: Record<string, ValueType>[];  // 数据行
  affectedRows: number;  // 影响行数 (INSERT/UPDATE/DELETE)
  executionTime: number; // 执行时间 (毫秒)
  error?: string;         // 错误信息
}
```
